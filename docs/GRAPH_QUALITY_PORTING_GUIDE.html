Porting Graph‑Quality into knowledge_rag

Lift entity deduplication, garbage‑node filtering, connectivity repair, and the ontology dependency out of the Context Graph fork — without taking the rest of it.

This page is the map. The full step‑by‑step guide (Markdown) has the code, copy commands, migration SQL, endpoint reference, and rollout checklist.

Why this is a clean lift. None of the four packages import the Context Graph engine class — they reach the graph, vector DB, and LLM only through injected arguments. Your Postgres backend already implements every graph method they call, and each store is a template‑method base class whose only backend surface is a JSONB blob. No changes to your LightRAG fork.

How it maps onto your stack

Two facts that make it clean

  • Your LightRAG fork is effectively vanilla — no extraction‑format mismatch; these features act on the resulting graph.
  • You're single‑tenant (kdefault_v1) — endpoints resolve the workspace server‑side; no workspace param/selector anywhere.

Decisions we made for you

  • Endpoints live in ingestion_server/main.py (it owns the write‑authority _rag).
  • Side‑stores become Postgres, not JSON files — matches your all‑PG model & read‑only UI mount.
  • Maintenance is an operator action → admin‑gated UI, not agent MCP tools.

The ten steps

1

2
Sever the ontology → rules edge write — copy two pure‑regex parsers into projection_min.py; change one import in ontology/schema.py. Bundle is now closed.
3
Postgres stores write — migrations/007 (one JSONB table) + a ~20‑line _PgBlob mixin implementing _read_raw/_write_raw. All store logic is inherited.
4
The QualityLightRAG subclass write — copy the on‑demand method bodies verbatim from core.py; inject via make_lightrag(lightrag_cls=…) + attach_quality(pool).
5
On‑ingest garbage filter — the subclass's _process_extract_entities override runs inside your existing ainsert; no worker.py change.
6
Config knobs — DEDUP_*/GARBAGE_* into the Config dataclass + compose env; optional app_settings live override (your recycle_days precedent).
7
HTTP endpoints write — plain @app.* handlers under /api/graph/*, mirroring _delete_one_with_lightrag. Start read‑only + previews.
8
Background sweep optional — drain the gray‑band review queue on an interval, mirroring your recycle sweep. Off by default.
9
NiceGUI page write — one pages/graph_quality.py + a few api_client methods + 3 edits in app.py. No workspace selector needed.
10
MCP diagnostics optional — expose only read‑only reports as agent tools; keep mutations operator‑only.
Operator caveats to surface in the UI. unmerge restores the alias/audit but does not re‑split folded edges (preview merges carefully). Garbage removal & isolate prune move nodes to quarantine (restorable), not hard delete. Only degree‑0 isolates are pruned.
