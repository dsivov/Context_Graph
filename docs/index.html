Context Graph

A decision-aware knowledge-graph system built on LightRAG. It extends triples (h, r, t) into contextual quadruples (h, r, t, rc) — attaching the decision record (who, why, when, under which policy) to every edge — and adds CGR3 reasoning, a business rules engine, a typed ontology, web ingestion, and a governed action layer.

Built on LightRAG (HKUDS) · source on GitHub · supported by Sellence
(h, r, t, rc) quadruples CGR3 reasoning rules & ontology web ingester MCP + REST multi-tenant

Governance

Business Rules Engine →

A pre-emit gate over every decision. A DSL with semantic sim() matching, PASS / FLAG / REJECT outcomes, and a plain-English policy author.
Typed schema

Ontology →

Object types, link types, and typed properties that every extraction is validated — and coerced — against. Open- or closed-world, with an NL schema author.
Ingestion

Web Ingester →

Point it at any site. A polite crawler, pluggable platform connectors, and LLM-guided acquisition — with the source URL carried through as provenance.
Execution

Action Layer →

Executable operations bound to object types. Invoke → rules gate → audit edge, with SSRF-guarded handlers — turning a graph you reason from into one you operate from.

Getting started
Set USE_CONTEXT_GRAPH=true, start lightrag-server (port 9621), and open the WebUI — the Rules and Ontology tabs manage governance, POST /scrape pulls a website into the graph, and POST /actions/invoke runs governed operations. Full setup lives in the README.
